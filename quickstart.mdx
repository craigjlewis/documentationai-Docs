## Overview

Spin up the `trading-app-web` stack locally: FastAPI backend, React/Vite frontend, and optional Streamlit UI and Celery workers.

## Prerequisites

<Callout kind="info">

Ensure you have:

- Python 3.10 or later
- Node.js 18 or later and npm
- Git
- Optional: Redis (for Celery tasks)
- Optional: OpenAI/Anthropic keys if you plan to use LLM features

</Callout>

Clone the repo into a local directory before starting:

```bash
git clone <REPO_URL>
cd trading-app-web
```

## 1. Create a virtual environment and install Python dependencies

Use a Python virtual environment to isolate dependencies.

<Steps>
  <Step title="Create and activate venv" icon="terminal">

    <CodeGroup tabs="bash (macOS/Linux),bash (Windows PowerShell)">
```bash
python3 -m venv .venv
source .venv/bin/activate
```
```bash
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```
    </CodeGroup>

    Your shell prompt should show `(.venv)` once the environment is active.
  </Step>

  <Step title="Install backend requirements" icon="download">

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

    After this completes, `pip list` should show FastAPI, Uvicorn, and other project dependencies.
  </Step>
</Steps>

## 2. Configure environment variables

Copy the example environment file and set minimum required variables.

<Steps>
  <Step title="Copy .env template" icon="file-plus">

```bash
cp .env.example .env
```

    This creates a local `.env` file that the backend reads at startup.
  </Step>

  <Step title="Set core variables" icon="settings">

Update `.env` and set at least:

- `ADMIN_USERNAME` to an admin login name, for example `admin`
- `ADMIN_PASSWORD` to a strong password
- `SECRET_KEY` to a long random string
- `DATABASE_URL` to the default SQLite URL if not already set: `sqlite+aiosqlite:///./data/trading.db`

  </Step>
</Steps>

## 3. Run the FastAPI backend

Start the API server with Uvicorn.

<Steps>
  <Step title="Start backend server" icon="play-circle">

```bash
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

    On success, the terminal shows logs with `Uvicorn running on http://0.0.0.0:8000`.
  </Step>

  <Step title="Verify health and docs" icon="check-circle">

Open these URLs in your browser:

- API docs: `http://localhost:8000/docs`
- Health check: `http://localhost:8000/health`

    You should see the interactive docs UI and a healthy status response.
  </Step>
</Steps>

## 4. Run the React/Vite frontend (primary UI)

Run the main web UI that talks to the FastAPI backend.

<Steps>
  <Step title="Install frontend dependencies" icon="download">

```bash
cd frontend
npm install
```

    Stay in the `frontend` directory for the next command.
  </Step>

  <Step title="Start Vite dev server" icon="play-circle">

```bash
npm run dev
```

    Vite prints a local URL, typically `http://localhost:5173`.
  </Step>

  <Step title="Open the app" icon="monitor">

    Visit `http://localhost:5173` in your browser.

    Keep both the backend (port 8000) and frontend (port 5173) running in separate terminals.
  </Step>
</Steps>

## 5. Optional: Run the Streamlit app (legacy/alt UI)

Use the legacy Streamlit dashboard if you want a quick alternative interface.

<Steps>
  <Step title="Start Streamlit server" icon="play-circle">

    From the project root with the virtual environment active, run:

```bash
streamlit run app.py
```

    Streamlit starts on `http://localhost:8501` by default.
  </Step>

  <Step title="Access the Streamlit UI" icon="monitor">

    Open `http://localhost:8501` to use the Streamlit-based trading dashboard alongside or instead of the React UI.
  </Step>
</Steps>

## 6. Optional: Run Celery workers for background tasks

If your workflows use background jobs, start Celery workers and beat. This requires a running Redis instance configured in `.env`.

<Callout kind="info">

Set a valid Redis URL in your `.env`, for example `REDIS_URL=redis://localhost:6379/0`, before starting Celery.

</Callout>

<Steps>
  <Step title="Start Celery worker" icon="zap">

```bash
celery -A celery_app worker --loglevel=info
```

    The terminal should show Celery connecting to Redis and waiting for tasks.
  </Step>

  <Step title="Start Celery beat (scheduler)" icon="clock">

```bash
celery -A celery_app beat --loglevel=info
```

    This schedules any periodic tasks defined in the project.
  </Step>
</Steps>

## Recap

At this point you should be able to:

- Hit `http://localhost:8000/docs` and `http://localhost:8000/health`
- Use the React/Vite UI at `http://localhost:5173`
- Optionally use the Streamlit UI at `http://localhost:8501`
- Optionally run Celery workers with Redis for background tasks

Keep the backend and your chosen UI running while you explore features like BrowserAct FX reports and automated trading workflows.
