## Understand the Reports section

Use the Reports section to access reporting, analytics, and scheduling capabilities from a central hub in the trading app.

<Callout kind="info">

The Reports sidebar group uses id `reports` with title `REPORTS`.  
Routing is configured in `frontend/src/App.tsx` (for example, mapping `/reports-hub` to the reports hub page).  
The app remembers which sidebar groups are expanded using the `sidebar-expanded-sections` key in `localStorage`.

</Callout>

## Routes and page components

The Reports area is currently implemented as a single hub route that loads a dedicated React page component.

- **Route path**: `/reports-hub`
- **React page component**: `frontend/src/pages/ReportsHub.tsx`

Use this route when adding navigation links, deep-linking from other parts of the app, or wiring up tests around the Reports UI.

## Backend services used by Reports

Reports features typically rely on service modules that wrap backend APIs and proxy calls under the `/api/...` namespace.

- **Reports API service**: `frontend/src/services/reportsApi.ts`  
  Likely handles listing, generating, and retrieving individual reports via proxied endpoints such as `/api/reports/...`.

- **Report schedules API service**: `frontend/src/services/reportScheduleApi.ts`  
  Likely manages creating, updating, and deleting report schedules via endpoints such as `/api/report-schedules/...`.

When you extend Reports functionality, prefer to add methods in these service modules instead of calling `/api/...` endpoints directly from components.

## Basic Reports workflow

Use this quick workflow when you wire up or test the Reports hub.

<Steps>

<Step title="Open the Reports hub" icon="folder">

Navigate in the sidebar to the **REPORTS** group and select **Reports Hub**, which routes to `/reports-hub` and renders `ReportsHub.tsx`. Confirm the URL updates and the hub layout loads without console errors.

</Step>

<Step title="Load available reports" icon="database">

From the hub, trigger the action that lists available reports. Verify that `reportsApi.ts` calls a proxied `/api/...` endpoint and that the UI renders the returned report list or an empty state.

</Step>

<Step title="Configure a report schedule" icon="settings">

Create or edit a report schedule. Confirm that `reportScheduleApi.ts` sends the appropriate request under `/api/...` and that the new schedule appears in the UI after a successful save.

</Step>

</Steps>
